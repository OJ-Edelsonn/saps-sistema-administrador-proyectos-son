# Guion de sustentacion

1. Presento S.A.P.S como una solucion para administrar proyectos integradores.
2. Explico el problema: seguimiento manual, observaciones dispersas y evaluaciones poco centralizadas.
3. Muestro la arquitectura MVC y las capas del proyecto.
4. Presento entidades y relaciones principales.
5. Demuestro login, dashboard y CRUDs.
6. Registro un proyecto, un entregable y una observacion.
7. Explico la evaluacion con rubrica y el calculo de nota.
8. Muestro reportes y cierro con conclusiones.
